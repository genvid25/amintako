# «ЭХО В ГОРАХ» — конвейер кадров

Система, которая превращает раскадровку мультфильма в готовые кадры. Промты и
картинки делает агент-генератор прямо в браузере (в залогиненных ChatGPT и Google
Labs под общими подписками команды), складывает результат в этот репозиторий, а
очередь кадров, статусы и правки ведёт в облачной базе. Режиссёр и аниматоры
смотрят кадры, одобряют их и правят промты в веб-дашборде. Прежний пилот с
генерацией через API отменён — его код лежит в `scripts/archive/` и запускать его
нельзя.

## Карта проекта

**Документы** (в корне):

- **ВОРКЕР.md** — инструкция агента-генератора: что он делает при каждом запуске
  (забор материалов, промты, генерация, публикация посценно, отчёт). Главный
  рабочий документ конвейера.
- **ПАМЯТКА-РЕЖИССЁРУ.md** — для режиссёра: как всё установить и как работать
  (загрузка материалов, проверка кадров, личные ссылки команде).
- **ПРОМТ-ГАЙД.md** — правила студии для англоязычных промтов. Источник правды по
  тому, как писать промт.

**База данных** — `docs/`:

- **supabase-schema.sql** — основная схема: таблицы очереди, функции, права.
  Выполняется в Supabase → SQL Editor **первой**.
- **supabase-storage-политики.sql** — режиссёрский раздел: загрузка материалов в
  бакет `materials`, создание серий и сотрудников. После схемы.
- **supabase-правка-промтов.sql** — дополнение: ручная правка промта в дашборде.
- **supabase-роль-режиссёра.sql** — дополнение: режиссёр решает по всем сериям.
- **supabase-план.md** — как всё устроено и почему (пояснительная записка, не SQL).

Порядок: сначала `supabase-schema.sql`, затем три дополнения
(`storage-политики`, `правка-промтов`, `роль-режиссёра`) — в любом порядке между
собой. Все скрипты безопасно запускать повторно.

**Сайт** — `dashboard/`:

- Дашборд кадров на GitHub Pages: **https://genvid25.github.io/amintako/dashboard/**
- По этой ссылке — только просмотр. Чтобы одобрять кадры и править промты, нужна
  личная ссылка `?key=…`; её выдаёт схема базы (см. ПАМЯТКА-РЕЖИССЁРУ.md).

**Скрипты** — `scripts/`:

- **fetch_materials.py** — забирает материалы режиссёра из облака в проект.
- **load_storyboard.py** — заливает раскадровку (сцены, кадры, версии) в базу.
- **archive/** — старый пилот генерации через API и локальный сервер дашборда.
  История, запускать не нужно (см. `scripts/archive/ПРОЧТИ.txt`).

## Где что лежит

- **Данные** (очередь кадров, статусы, версии, правки, материалы) — в облаке
  **Supabase**. Это источник правды.
- **`data/storyboard.json`, `data/frames.json`** — локальная раскадровка и промты,
  из которых `scripts/load_storyboard.py` заливает серию в базу.
- **Картинки** — в этом репозитории (`images/`, `previews/`), раздаются через
  GitHub Pages. Исходные PNG в репозиторий не попадают.
