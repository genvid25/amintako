# Гайд: промты для nano banana Pro (gemini-3-pro-image) — кадры мультфильма в стиле 3D-анимации

Источники: официальные гайды Google (ai.google.dev/gemini-api/docs/image-generation, deepmind.google/models/gemini-image/prompt-guide, blog.google prompting-tips-nano-banana-pro, cloud.google.com ultimate-prompting-guide) + практика сообщества + **инструкция команды по NB Pro (Claude.mhtml, проект «Путь Амина»)**. Составлен 16.07.2026, обновлён после инструкции команды.

## Главный принцип
NB Pro получает ТРИ источника информации: (1) фото-ингредиенты персонажей — лицо, тело, одежда; (2) фото-ингредиент локации — интерьер, мебель, свет; (3) текст промта — действие, эмоция, ракурс камеры, свет, стиль.
**Золотое правило: текст НЕ описывает то, что уже есть на фото.** Не описывай внешность персонажей и детали локации — модель берёт их из ингредиентов. Текст описывает только действие/эмоцию/камеру/свет/стиль.
Модель рассуждает и выполняет промт буквально (не переписывает и не «додумывает», как ChatGPT) — пиши связные предложения, не теги (`boy, boat, 4k, masterpiece` — запрещено). Запросы stateless: ингредиенты и identity lock повторяются В КАЖДОМ кадре.

## НИКОГДА не описывать в тексте (берётся из фото)
- Внешность: цвет волос, цвет глаз, рост, тип тела.
- Возраст/пол словами: boy, girl, child, young, old, he, she. Субъект = «the exact character in [одежда]» — без имени и без местоимений.
- Детали локации/интерьера: мебель, стены, объекты-декорации.
- Экранное позиционирование: «on the left / on the right / in the center». (Относительное действие — «sits across the table», «leaning over the bench» — можно и нужно.)
- Погоду/время суток, которых НЕТ в фото локации: промтом их надёжно не добавить и не убрать — это правится в самом фото локации (эдит PIL/NumPy).
Эмоцию и мимику описывать МОЖНО и нужно (brow furrowed, eyes shining) — это не «внешность».

## Ingredients (ингредиенты-референсы)
| Тип сцены | Формула |
|---|---|
| 1 персонаж | 1 char + 1 location = 2 |
| 2 персонажа | 2 char + 1 location = 3 |
| 3 персонажа | 3 char + 1 location = 4 |

**Максимум 4 ингредиента** — больше даёт нестабильный результат. Если в кадре >3 персонажей: одного опиши текстом (без ингредиента) или упрости композицию. (Официальный потолок Google выше — консистентность до 5 персонажей / до 14 изображений — но рабочий предел NB Pro на практике = 4 ингредиента.)
- Локация — ВСЕГДА ингредиент.
- Порядок ингредиентов фиксированный: персонажи (в порядке упоминания), затем локация. Привязка — по ОДЕЖДЕ, не по голому номеру («in a grey hoodie»); Image N — лишь слот ингредиента.
- Прикладывай только тех, кто в кадре. Меньше персонажей — стабильнее вывод.

## Identity Locking (закрепление персонажа) — важнейшая техника
Персонаж должен быть упомянут в первых 10 словах промта. Одежда обязательна при КАЖДОМ упоминании — это сильный идентификатор.
- Первое упоминание: `The exact character in [одежда] from Image N, preserving identical facial features and proportions, [действие + эмоция].`
- Повторное: `the exact character in [одежда] from Image N, [действие + эмоция].`
Слабо: «The exact character from Image 1». Сильно: «The exact character in a grey hoodie from Image 1».
Для 2-3 персонажей — цепочка `First… Then… Finally…` (по одному персонажу за раз).

## Скелет промта NB Pro
`[IDENTITY + ACTION + EMOTION] → [LOCATION ref] → [CAMERA + LIGHTING] → [CLOSING BLOCK]`

1. **Identity + действие + эмоция** (перем.): identity lock (см. выше) + что делает + мимика. Настоящее действие, одно на кадр.
2. **Локация** (ФИКС. на сцену): `Use Image N as the exact background location.`
3. **Камера + свет** (камера перем., свет ФИКС. на сцену): один ракурс + один описатель света.
4. **Closing block — ФИКС. ДОСЛОВНО НА ВСЮ СЕРИЮ:**
   `No dust in the air. No text, no watermarks. High quality, vivid colors, sharp details, cinematic lighting. 3D animation, cinematic.`

Длина: **60-80 слов** без closing-блока. Не перегружай. Для 2-3 персонажей промт длиннее (цепочка) — это нормально.

## Камера: крупность раскадровки → термин
| Раскадровка | В промте | Что в кадре |
|---|---|---|
| ОБЩИЙ | wide shot | полная локация, персонажи мелко |
| СРЕДНИЙ | medium shot (основной рабочий ракурс) / medium-wide shot (персонаж + окружение) | поясной |
| «по грудь» | medium close-up | грудь + лицо |
| КРУПНЫЙ | close-up | лицо, эмоция |
| ДЕТАЛЬ | extreme close-up / insert shot | рука, глаза, предмет |

Модификаторы угла: eye-level, low angle, high angle, slight high angle.

## Свет: один описатель, единый по сцене
- `bright daylight` — дневные сцены
- `warm evening light` — вечер
- `dramatic side lighting` — напряжённые моменты
- `soft diffused light` — спокойные сцены

Свет должен соответствовать фото локации и держаться одинаковым внутри сцены.

## Реквизит
Упоминай ТОЛЬКО если персонаж взаимодействует. OK: «hand on the tuning dial of a radio». НЕ OK: «a radio on the bench» (просто стоит — модель может задублировать или выдумать предмет).

## Стиль
Только `3D animation, cinematic` (входит в closing block). **НЕ писать** «3D Pixar-style», «Pixar», «family-friendly» / «семейный контент». Запрещённые слова (ломают анимационный look): photo, photograph, photorealistic, realistic skin, hyperrealistic, uncanny, 8k photo.

## Верность сценарию (7 правил) — для массового написания промтов
1. Масштаб точно по тексту — не преувеличивай; нет слова «огромный» → не пиши enormous.
2. Эмоция — нюанс из сценария, не ярлык.
3. Действия буквально из сценария.
4. Реквизит только из сценария — не добавляй фантазийных предметов.
5. Не додумывай эмоции — нет «улыбается» в тексте → не пиши smiles.
6. Сохраняй драматургическую функцию кадра в истории.
7. Тон — из контекста сцены.

## Чеклист (перед выдачей промта)
1. Identity lock в первых 10 словах; одежда указана при каждом упоминании персонажа.
2. НЕТ описания внешности / интерьера / экранного позиционирования / возраста-пола / местоимений.
3. Погода и время суток — только те, что есть в фото локации (иначе править фото, не текст).
4. ≤4 ингредиента (3 char + 1 location). >3 персонажей → одного текстом или упростить.
5. Локация — ингредиент: `Use Image N as the exact background location`.
6. Один ракурс камеры + один описатель света (свет единый на сцену).
7. Эмоция из реплики → мимика/поза. Текст реплики в кадр НЕ попадает никогда.
8. Реквизит — только при взаимодействии.
9. Цепочка First… Then… Finally… для 2+ персонажей.
10. Closing block дословно (No dust… 3D animation, cinematic).
11. Позитивные формулировки — описывать что ЕСТЬ (называние нежелательного провоцирует его появление). Параметра negative prompt нет; короткий фикс-хвост про пыль/текст — единственные негативы.
12. 60-80 слов (без closing-блока).
13. Стиль: «3D animation», без «Pixar»/«family-friendly».
14. 16:9 и 2K — в КОНФИГЕ (`imageConfig: {"aspectRatio": "16:9", "imageSize": "2K"}`, K заглавная), не текстом.
15. Один кадр = один независимый запрос со своими ингредиентами и identity lock (память сессии не работает).

## Ограничения NB Pro (правится НЕ промтом)
- Погода / снег / свет из референса: промтом не переопределить — редактируй фото локации (PIL/NumPy).
- Часы: точное время не рендерится надёжно — накладывать в пост-продакшене.

## Пример 1 — крупный план, эмоция надежды (Тако)
```
The exact character in a red-and-white striped shirt and red cap from Image 1, preserving identical facial features and proportions, grips the wooden edge of a small boat, chin lifted, eyes wide and shining with fragile hope, a faint hopeful smile emerging. Use Image 2 as the exact background location. Tight close-up, eye-level, shallow depth of field. Dramatic side lighting.
No dust in the air. No text, no watermarks. High quality, vivid colors, sharp details, cinematic lighting. 3D animation, cinematic.
```
Ингредиенты: Image 1 = Тако, Image 2 = локация (штормовое море ночью — заложено в самом фото). Итого 2 ингредиента.

## Пример 2 — общий план, лодка с 3 героями, вдали корабль-призрак в тумане
```
First, the exact character in a grey hoodie from Image 1, preserving identical facial features and proportions, sits at the front of a small boat, leaning forward and pointing ahead, eyes narrowed with tense focus. Then, the exact character in a red-and-white striped shirt and red cap from Image 2, preserving identical facial features, grips the wooden edge, mouth open in a gasp. Finally, the exact character in a black hoodie from Image 3, preserving identical facial features, pulls the oar, jaw set with effort. Use Image 4 as the exact background location. Wide shot, low angle, deep focus. Soft diffused light.
No dust in the air. No text, no watermarks. High quality, vivid colors, sharp details, cinematic lighting. 3D animation, cinematic.
```
Ингредиенты: Image 1 = Амин, 2 = Тако, 3 = Карим, 4 = локация (море в тумане + силуэт корабля-призрака заложены в фото). Итого 4 ингредиента (максимум). Корабль-призрак и туман — в фото локации, НЕ в тексте.

## Отличия от ChatGPT-промтов (мигрируем с ChatGPT)
- ChatGPT тайно расширяет короткий промт — NB Pro выполняет текст буквально: полное описание пишем сами.
- Канон внешности = ингредиенты + одежда (identity lock), а НЕ словесный портрет. Имена и местоимения в промте не нужны.
- Хвосты «masterpiece, 8k, trending…» бесполезны и вредят.
- Негативов нет — только позитивное описание + короткий фикс-хвост (No dust, No text).
- Формат — через конфиг, не через текст.
- «Память сессии» не работает: каждый кадр — независимый запрос со своими ингредиентами и identity lock.

## Учтено из инструкции команды (Claude.mhtml)
Принятые правки (инструкция специфична для NB Pro и в этих пунктах конкретнее — она выигрывает):
1. **Золотое правило «текст не описывает то, что на фото»** + список «никогда не описывать» (внешность, возраст/пол, интерьер, экранное позиционирование). Убрали из скелета словесные описания внешности персонажей.
2. **Identity lock по ОДЕЖДЕ**: «The exact character in [одежда] from Image N, preserving identical facial features and proportions», в первых 10 словах, одежда при каждом упоминании. Заменили прежнее «featuring [Имя]… keep face identical».
3. **Максимум 4 ингредиента** (3 char + 1 location) как рабочий предел вместо теоретических 5/14; >3 персонажей → одного текстом или упростить.
4. **Цепочка First… Then… Finally…** для 2-3 персонажей.
5. **Closing block дословно**: «No dust in the air. No text, no watermarks. High quality, vivid colors, sharp details, cinematic lighting. 3D animation, cinematic.» («No dust» — их фикс от дымки/пыли, которую NB Pro любит добавлять).
6. **Стиль «3D animation», БЕЗ «3D Pixar-style» и «family-friendly»** — по новейшему указанию команды (Memory), которое отменяет старое «3D Pixar-style» в их же документе.
7. **Реквизит — только при взаимодействии персонажа.**
8. **Свет — один описатель из набора** (bright daylight / warm evening light / dramatic side lighting / soft diffused light), единый по сцене.
9. **Расширенная таблица крупности** (medium shot, medium-wide, medium close-up + модификаторы угла).
10. **7 правил верности сценарию** (масштаб, эмоция, действия, реквизит — буквально из текста, ничего не додумывать).
11. **Ограничения NB Pro**: погода/свет из референса и часы правятся не промтом (эдит фото / пост-продакшен).
12. **Длина 60-80 слов**; без экранного позиционирования и местоимений.

Отклонено (не относится к нашему конвейеру стилл-кадров):
- Весь блок VEO / Seedance и звук / lip-sync / тайм-коды — это ВИДЕО.
- First/Last frame + Delta Rule — видео-пайплайн (пара ключевых кадров для интерполяции VEO); у нас один стилл на кадр раскадровки. Опционально: приём «Maintain exact visual continuity with Image N» пригодится, только если два соседних кадра — один сетап и предыдущий кадр подаётся ингредиентом.
- Старое «3D Pixar-style, family-friendly» — отменено самим заказчиком в Memory.
- Пример с 5 персонажами / 6 ингредиентами из их документа — противоречит новому правилу «максимум 4».
- Формат вывода «в чат, не файлами» — чат-воркфлоу, к нашему API-конвейеру не относится.
